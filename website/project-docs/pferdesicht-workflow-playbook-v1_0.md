# WORKFLOW-PLAYBOOK v1.0
## Forward-Engineered aus 6 Wochen Pferdesicht-Projekt
### Wiederverwendbare Workflows für jedes neue Nischen-Content-Projekt

---

**Zweck:** Bei einem neuen Projekt sofort parallel Workflows anstoßen statt alles nochmal von vorne überlegen.

**Wie nutzen:** Workflow auswählen → Deep Research Prompts abfeuern (Gemini/Claude) → Ergebnisse konsolidieren → Framework anwenden

## WF-01: ZIELGRUPPEN-DEEP-DIVE

### Was du bekommst:
- Vollständige ICP-Definition (Ideal Customer Profile)
- 3-Schichten-Modell (sagt / fühlt / verschweigt)
- VOC-validierte Sticky Messages
- Urängste und Kaufmotivationen
- Empowerment-Framework

### Bewährt bei Pferdesicht:
Anna-Empowerment-Framework v2.6 → 17 Seiten, 7 Versionen, Grundlage für ALLES

### Deep Research Prompt (→ Gemini):

```
AUFTRAG: Erstelle eine vollständige Zielgruppen-Analyse für [NISCHE/PROJEKT].

RECHERCHIERE:
1. DEMOGRAFISCH: Alter, Geschlecht, Einkommen, Bildung, Region
2. PSYCHOGRAFISCH: Werte, Ängste, Frustrationen, Wünsche, Selbstbild
3. VERHALTEN: Wo suchen sie Infos? Welche Plattformen? Welche Influencer?
4. SPRACHE: Originalzitate aus Foren, Amazon-Rezensionen, Facebook-Gruppen, Reddit
   → Suche mindestens 20 echte Zitate die zeigen WIE die Zielgruppe redet
5. SCHMERZPUNKTE: Die Top-5 Probleme, sortiert nach Dringlichkeit
6. INFORMATIONS-PARADOX: Wie ist die Zielgruppe überinformiert aber nicht wissend?
7. KAUFAUSLÖSER: Was muss passieren damit jemand kauft? (Trigger-Events)
8. EINWÄNDE: Die 5 häufigsten Gründe NICHT zu kaufen

SPEZIELL - DAS 3-SCHICHTEN-MODELL:
Für jeden Schmerzpunkt definiere:
- SCHICHT 1 (sagt offen): Was die Person im Forum/in der Beratung sagt
- SCHICHT 2 (fühlt): Die Emotion dahinter (Angst, Scham, Frustration)  
- SCHICHT 3 (verschweigt): Der tiefste Kern den niemand ausspricht

OUTPUT-FORMAT:
- Persona-Profil (1 Seite)
- 3-Schichten-Tabelle pro Schmerzpunkt
- 20+ echte Zitate mit Quelle
- Kaufauslöser-Hierarchie
- Einwand-Entkräftungs-Matrix

QUELLEN:
- Reddit-Threads, Facebook-Gruppen, Amazon-Rezensionen
- Branchen-Foren, YouTube-Kommentare
- Studien und Marktforschung wenn verfügbar
- Google Trends, Answer the Public
```

### Nach der Recherche — Konsolidierung:

```
KONSOLIDIERE die Deep Research zu einem ICP-Framework nach diesem Muster:

1. PERSONA-NAME + BESCHREIBUNG (wie "Anna — Die beschützende Anwältin")
2. DAS 3-SCHICHTEN-MODELL (Tabelle mit allen Schmerzpunkten)
3. STICKY MESSAGES (echte Zitate, validiert, nach Kategorie sortiert)
4. URÄNGSTE (die tiefsten Ängste die Schicht 3 speisen)
5. INFORMATIONS-DILEMMA (wie ist die Zielgruppe überinformiert?)
6. EMPOWERMENT-DEFINITION (was ist der "Quiet Luxury" Moment?)
7. CONTENT-ANWEISUNG PRO SCHICHT (was darf angesprochen werden, was nicht)

Dieses Framework wird die Grundlage für ALLE Content-Entscheidungen.
```

### Erfolgskriterium:
Du kannst für jeden Content-Entwurf sofort prüfen: "Spricht das Schicht 1, 2 oder 3 an? Ist das erlaubt?"

## WF-02: WETTBEWERBS-ANALYSE + POSITIONIERUNG

### Was du bekommst:
- Kontroverser Glaubenssatz (Polarizing Belief)
- Unique Value Proposition
- Positionierungs-Matrix (du vs. Wettbewerb)
- Golden Circle (WHY/HOW/WHAT)

### Deep Research Prompt (→ Gemini):

```
AUFTRAG: Wettbewerbs-Analyse und Positionierung für [NISCHE/PROJEKT]

RECHERCHIERE:
1. TOP 10 WETTBEWERBER: Wer bedient diese Zielgruppe bereits?
   → Blogs, Kurse, YouTube-Kanäle, Instagram-Accounts, Podcasts
   → Für jeden: Stärke, Schwäche, Preispunkte, Tonalität
2. CONTENT-LÜCKEN: Was fehlt im Markt? Was macht KEINER gut?
3. PREIS-LANDSCHAFT: Was kosten vergleichbare Produkte?
4. VERTRAUENS-SIGNALE: Was nutzen erfolgreiche Wettbewerber? (Zertifikate, Testimonials, etc.)
5. POSITIONIERUNGS-MÖGLICHKEITEN: Wo ist PLATZ für etwas Neues?

SPEZIELL:
- Identifiziere einen KONTROVERSEN GLAUBENSSATZ der die Zielgruppe spaltet
  (Beispiel Pferdesicht: "Pferdebesitzer die 'zu viel' fragen sind die Lösung, nicht das Problem")
- Definiere den GOLDEN CIRCLE:
  WHY: Warum existiert dieses Projekt?
  HOW: Was macht es ANDERS als alle anderen?  
  WHAT: Was wird konkret angeboten?

OUTPUT-FORMAT:
- Wettbewerber-Matrix (10 Spalten)
- Content-Lücken-Analyse
- Positionierungs-Statement (1 Satz)
- Kontroverser Glaubenssatz + warum er funktioniert
- Golden Circle Visualisierung
```

## WF-03: VOICE & BRAND PERSONALITY

### Was du bekommst:
- Voice-Guide (wie spricht die Marke?)
- Küchentisch-Test adaptiert
- Do's und Don'ts für Tonalität
- Diana-Bibel-Equivalent (Tracking-System)
- Scham-Formel für sensible Themen

### Deep Research Prompt (→ Gemini):

```
AUFTRAG: Voice & Brand Personality für [NISCHE/PROJEKT]

Der/die Gründer:in ist [NAME] mit [X] Jahren Erfahrung in [BEREICH].

RECHERCHIERE:
1. MARKT-TONALITÄT: Wie sprechen die erfolgreichsten Creator in dieser Nische?
2. ZIELGRUPPEN-ERWARTUNG: Welchen Ton erwartet die Zielgruppe? (Experte vs. Freund vs. Coach)
3. DACH-BESONDERHEITEN: Was funktioniert im deutschen Markt anders als im US-Markt?
4. VERTRAUENSAUFBAU: Wie wird in dieser Nische Vertrauen aufgebaut? (Studien vs. Erfahrung vs. Ergebnisse)
5. SENSIBLE THEMEN: Welche Themen lösen Scham oder Schuld aus bei der Zielgruppe?

ERSTELLE:
- BRAND VOICE GUIDE mit 5 Adjektiven die den Ton beschreiben
- KÜCHENTISCH-TEST: "Würde [NAME] das so zu [PERSONA] am Küchentisch sagen?"
- DO'S AND DON'TS Tabelle (was die Marke tut / nie tut)
- SCHAM-FORMEL für sensible Themen:
  1. Externalisieren (nicht "du machst falsch" sondern "man lernt das nirgends")
  2. Normalisieren ("die meisten machen das so")
  3. Validieren ("dein Instinkt ist richtig")
  4. Eigene Fehler teilen
  5. DANN erst informieren
- TRACKING-SYSTEM: Vorlage zum Tracken verwendeter Metaphern, Stories, Fakten
```

## WF-04: CONTENT-ARCHITEKTUR + PYRAMIDE

### Was du bekommst:
- Content-Pyramide (Social → Blog → Lead Magnet → Produkt)
- Themen-Cluster mit Cross-Referenzen
- Ableitungsmatrix (was darf auf welcher Ebene stehen)
- Content-Volumen-Planung

### Bewährt bei Pferdesicht:
G18-Content-Ableitungsmatrix → 4 Module = 14 Blog-Artikel + 57 Social Posts + 10-Seiten Leadmagnet

### Konsolidierungs-Prompt:

```
Basierend auf [ICP-Framework] und [Positionierung]:

ERSTELLE eine Content-Architektur mit diesen Ebenen:

EBENE 4 — PRODUKT (€XX): Was ist die KOMPLETTE Transformation?
EBENE 3 — LEADMAGNET (€0): Was ist der QUICK WIN + E-Mail?
EBENE 2 — BLOG: Was baut VERTRAUEN + SEO auf?
EBENE 1 — SOCIAL MEDIA: Was weckt AUFMERKSAMKEIT?

FÜR JEDES THEMA/MODUL definiere:
- Was steht auf welcher Ebene? (Ableitungsmatrix)
- Was verweist wohin? (Content-Treppe)
- Wie viele Ableitungen pro Modul? (Volumen-Planung)

GRUNDPRINZIP:
Social = 1 Fakt / 1 Mythos / 1 Frage → Blog
Blog = Das WARUM verstehen (1 Aspekt) → Leadmagnet
Leadmagnet = Das WAS TUN (Checkliste, Plan) → Produkt
Produkt = Das KOMPLETTE Bild (alle Sonderfälle)

OUTPUT: Content-Pyramide + Ableitungsmatrix pro Thema
```

## WF-09: PREMIUM-CONTENT ERSTELLEN

### Der 7-Phasen-Workflow (bewährt bei 5 G18-Modulen):

```
PHASE 1: BRIEFING & SCOPE
→ Thema in 1 Satz, Persona-Kernfrage, Fehlannahme, persönliche Geschichte, Abgrenzung

PHASE 2: STRUKTUR
→ Gliederung nach bewährtem Muster, Review, Freigabe

PHASE 3: RECHERCHE
→ Primärquellen, Originalstudien, Widersprüche klären, Quellenverzeichnis

PHASE 4: DRAFT
→ Alle Abschnitte schreiben, Tools/Tabellen, persönliche Geschichten

PHASE 5: VERIFIKATION
→ Experten-Review, Fakten-Check, NUR beauftragte Änderungen

PHASE 6: FEINSCHLIFF
→ Inline-Zitate, Quellenverzeichnis, DACH-Anpassungen

PHASE 7: FINALISIERUNG
→ Projektordner, Tracking-System updaten, Ableitungen planen
```

### Starter-Prompt für neues Modul:

```
BRIEFING: [PRODUKT] MODUL [X] — [THEMA]

Kontext:
- Projekt: [NAME]
- ICP: [Persona-Name und Kern-Beschreibung]
- Bisherige Module: [Liste]
- Voice: [Voice-Guide Referenz]

Modul [X] Scope:
- Thema: [THEMA]
- Kernpunkte: [3-5 Punkte]

Erste Aufgabe:
Lies den 7-Phasen-Workflow und starte mit Phase 1: Briefing & Scope.
Stelle die 5 Startfragen.
```

# MCP-AUTOMATION-POTENZIAL

### Was per MCP/Workflow automatisiert werden kann:

| Workflow | Manuell heute | Mit MCP/Agent |
|----------|--------------|---------------|
| WF-01 ICP | Deep Research Prompt kopieren → Gemini → Ergebnis kopieren → konsolidieren | Agent feuert Prompt ab → sammelt Ergebnis → konsolidiert automatisch |
| WF-05 Website | Design System manuell → Code in CC | Agent liest Design System → generiert Komponenten → deployed |
| WF-09 Content | Briefing → 7 Phasen manuell durchlaufen | Agent orchestriert Phasen, holt Recherche, liefert Draft |
| WF-10 Blog | Manuell aus Modul ableiten | Agent kennt Ableitungsmatrix → generiert Blog-Outlines automatisch |
| WF-11 Quiz | Fragen manuell designen | Agent kennt ICP → generiert Fragen + Scoring-Logik |
| WF-13 ManyChat | Flows manuell in ManyChat bauen | Agent generiert Flow-Konfiguration → Import in ManyChat |

### MCP-Server die relevant werden:
- **Vercel MCP**: Deploy-Automation, Preview-URLs, Environment Variables
- **Supabase MCP**: Datenbank für Quiz-Ergebnisse, User-Segmente, E-Mail-Listen
- **Canva MCP**: Slide-Generierung aus Content, Bulk-Export

### Gemini-Integration über MCP:
Sobald Gemini MCP-kompatibel wird (oder über API):
- Deep Research Prompts automatisch senden
- Ergebnisse automatisch in Projektordner speichern
- Konsolidierungen automatisch triggern
